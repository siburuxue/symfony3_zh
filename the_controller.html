控制器（Controller）¶

在前两个部分之后仍然在这？你已经变成了一个Symfony粉丝！无需再费周折，发现控制器可以为你做什么。

返回原始相应¶

Symfony将自己定义为一个请求响应式的框架。 当用户向你的应用程序发送一个请求时，Symfony创建一个Request对象来封装所有重新加载到改签请求的信息。 类似地，执行任何控制器的任何action结果是创建一个Response对象，Symfony用来生成返回给用户的html内容

到目前为止，本教程所有的操作都使用$this->render()的方式来返回相应的结果。 如果有必要，你可以创建原始的Response对象来返回任何文本内容： 

// src/AppBundle/Controller/DefaultController.php
namespace AppBundle\Controller;

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;
use Symfony\Bundle\FrameworkBundle\Controller\Controller;
use Symfony\Component\HttpFoundation\Response;

class DefaultController extends Controller
{
    /**
     * @Route("/", name="homepage")
     */
    public function indexAction()
    {
        return new Response('Welcome to Symfony!');
    }
}

路由参数¶

大多数情况下，应用程序的URL包含可变部分。 如果您正在创建博客这样的应用程序，则显示文章的网址应包含起标题或其他唯一标识符，以便让应用程序知道要显示的确切文章。

在Symfony的应用程序中，路径的变量部分用大括号括起来。（例如：/blog/read/{article_title}/）。 每个变量部分都被分配一个唯一的名字，以后可以在控制器中使用来检索每个值。

让我们用路由变量创建一个新的action来显示这个功能。 打开src/AppBundle/Controller/DefaultController.php文件并添加一个helloAction方法，其中包含以下内容： 

// src/AppBundle/Controller/DefaultController.php
namespace AppBundle\Controller;

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;
use Symfony\Bundle\FrameworkBundle\Controller\Controller;

class DefaultController extends Controller
{
    // ...

    /**
     * @Route("/hello/{name}", name="hello")
     */
    public function helloAction($name)
    {
        return $this->render('default/hello.html.twig', array(
            'name' => $name
        ));
    }
}

打开浏览器并访问http://localhost:8000/hello/fabien URL来查看执行这个新的action的结果。 代替这个action的结果的是你将看到的错误的页面。 正如你猜的那样，我们正在渲染一个还不存在的模板(default/hello.html.twig)。

用以下内容创建一个新的app/Resources/views/default/hello.html.twig模板： 

{# app/Resources/views/default/hello.html.twig #}
{% extends 'base.html.twig' %}

{% block body %}
    <h1>Hi {{ name }}! Welcome to Symfony!</h1>
{% endblock %}

再次浏览http://localhost:8000/hello/fabien网址,你将会看到这个新的模板呈现了控制器传递的信息。 如果您更改网址的最后一部分(例如 http://localhost:8000/hello/thomas)，并重新加载浏览器，页面将显示不同的信息。 如果你删除了网址的最后一部分(例如 http://localhost:8000/hello)，Symfony会显示错误，因为路由希望有一个名字，而你没有提供它。

使用Format¶

如今，一个Web应用程序不仅仅能提供html页面。从xml格式的RSS或者Web Service,到JSON格式的ajax请求，有很多不同的格式可供选择。 在Symfony中支持这些格式很直接，这要归功于一个叫做_format的特殊变量，它存储了用户请求的格式。

通过添加一个新的_format变量并且html作为默认值来调整hello路由。 

// src/AppBundle/Controller/DefaultController.php
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Template;

// ...

/**
 * @Route("/hello/{name}.{_format}", defaults={"_format"="html"}, name="hello")
 */
public function helloAction($name, $_format)
{
    return $this->render('default/hello.'.$_format.'.twig', array(
        'name' => $name
    ));
}

很显然，当你支持某种服务请求的格式时，你必须为每种支持的格式提供一个模板。 在这种情况下，你应该创建一个新的hello.xml.twig模板。 

<!-- app/Resources/views/default/hello.xml.twig -->
<hello>
    <name>{{ name }}</name>
</hello>

现在当你浏览http://localhost:8000/hello/fabien，显示是正常的，应为html是默认格式。 当你浏览http://localhost:8000/hello/fabien.html时，你依然会看到html页面，因为这次你指定了html页面。 最后，如果你访问http://localhost:8000/hello/fabien.xml，你会看到在浏览器中呈现的xml模板。

就这么简单。对于标准格式，Symfony也会自动为响应提供更好的Content-Type响应头。 要限制给定动作所支持的格式，请使用 @Route()注释的requirements选项。 

// src/AppBundle/Controller/DefaultController.php
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Template;

// ...

/**
 * @Route("/hello/{name}.{_format}",
 *     defaults = {"_format"="html"},
 *     requirements = { "_format" = "html|xml|json" },
 *     name = "hello"
 * )
 */
public function helloAction($name, $_format)
{
    return $this->render('default/hello.'.$_format.'.twig', array(
        'name' => $name
    ));
}

这个hello action现在将匹配/hello/fabien.xml或者/hello/fabien.json的网址， 但是如果您尝试获取/hello/fabien.js的网址，它将显示404错误，因为_format的值不符合要求。

重定向¶

如果您想将用户重定向到另一个页面，请使用redirectToRoute()方法： 

// src/AppBundle/Controller/DefaultController.php
class DefaultController extends Controller
{
    /**
     * @Route("/", name="homepage")
     */
    public function indexAction()
    {
        return $this->redirectToRoute('hello', array('name' => 'Fabien'));
    }
}

redirectToRoute() 方法将路由名称和可选的参数数组作为参数，并将用户重定向到使用这些参数生成的URL。

显示错误页面¶

在每个Web应用程序的执行过程中，错误都将不可避免地发生。 在出现404错误的情况下， Symfony包含一个可以在控制器中使用的快捷方便的方法： 

// src/AppBundle/Controller/DefaultController.php
// ...

class DefaultController extends Controller
{
    /**
     * @Route("/", name="homepage")
     */
    public function indexAction()
    {
        // ...
        throw $this->createNotFoundException();
    }
}

对于500错误，只要在控制器内部抛出一个普通的PHP异常， Symfony就会把它转换成适当的500错误页面： 

// src/AppBundle/Controller/DefaultController.php
// ...

class DefaultController extends Controller
{
    /**
     * @Route("/", name="homepage")
     */
    public function indexAction()
    {
        // ...
        throw new \Exception('Something went horribly wrong!');
    }
}

从请求中获取信息¶

有时您的控制器需要访问与用户请求有关的信息，例如他们的首选语言，IP地址或URL查询参数。 要访问这些信息，请在操作中添加一个类型为Request的新参数 这个新参数的类型名称并不重要，但是为了工作，它必须在Request类型的前面（不要忘记添加use语句导入这个新的Request类）： 

// src/AppBundle/Controller/DefaultController.php
namespace AppBundle\Controller;

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;
use Symfony\Bundle\FrameworkBundle\Controller\Controller;
use Symfony\Component\HttpFoundation\Request;

class DefaultController extends Controller
{
    /**
     * @Route("/", name="homepage")
     */
    public function indexAction(Request $request)
    {
        // is it an Ajax request?
        $isAjax = $request->isXmlHttpRequest();

        // what's the preferred language of the user?
        $language = $request->getPreferredLanguage(array('en', 'fr'));

        // get the value of a $_GET parameter
        $pageName = $request->query->get('page');

        // get the value of a $_POST parameter
        $pageName = $request->request->get('page');
    }
}

在模板中，您还可以通过Symfony自动提供的特殊app.request变量来访问Request对象： 

{{ app.request.query.get('page') }}

{{ app.request.request.get('page') }}

会话中的持久数据¶

即使HTTP协议是无状态的，Symfony也提供了一个很好的会话对象来表示客户端（不管是使用浏览器，机器人还是Web服务）。 在两个请求之间，Symfony通过使用本地PHP会话将属性存储在cookie中。

从任何控制器可轻松实现从会话中存储和检索信息： 

use Symfony\Component\HttpFoundation\Request;

public function indexAction(Request $request)
{
    $session = $request->getSession();

    // store an attribute for reuse during a later user request
    $session->set('foo', 'bar');

    // get the value of a session attribute
    $foo = $session->get('foo');

    // use a default value if the attribute doesn't exist
    $foo = $session->get('foo', 'default_value');
}

您也可以存储下一个请求后自动删除的“Flash消息”。 在将用户重定向到另一个页面（然后将显示该消息）之前，需要设置成功消息时，它们非常有用： 

public function indexAction(Request $request)
{
    // ...

    // store a message for the very next request
    $this->addFlash('notice', 'Congratulations, your action succeeded!');
}

你可以在模板中显示这样的flash消息： 

<div>
    {{ app.session.flashbag.get('notice') }}
</div>
