安装和配置Symfony¶

本章的目标是通过构建在Symfony之上的工作应用程序来启动和运行。 为了简化创建新应用程序的过程，Symfony提供了一个必须在创建第一个应用程序之前安装的安装程序。

Symfony安装工具¶

使用Symfony安装程序是创建新的Symfony应用程序的唯一推荐方式。 这个安装程序是一个只能安装一次的PHP应用程序，然后它可以创建任意数量的Symfony应用程序。

注解

安装程序需要PHP 5.4或更高版本。 如果您仍然使用旧版PHP 5.3版本，则不能使用Symfony安装程序。 阅读“不使用安装工具创建应用”部分，了解如何继续。

根据您的操作系统，安装程序必须以不同的方式安装。

Linux和Mac OS X系统¶

打开您的命令控制台并执行以下三个命令：

$ curl -LsS http://symfony.com/installer > symfony.phar
$ sudo mv symfony.phar /usr/local/bin/symfony
$ chmod a+x /usr/local/bin/symfony

这将在您的系统中创建一个全局symfony命令，用于创建新的Symfony应用程序。

Windows系统¶

打开您的命令控制台并执行以下命令：

c:\> php -r "readfile('http://symfony.com/installer');" > symfony.phar

然后，将下载的symfony.phar文件移动到您的项目目录并执行它，如下所示：

c:\> move symfony.phar c:\projects
c:\projects\> php symfony.phar

创建Symfony应用¶

Symfony安装程序准备就绪后，使用新命令创建您的第一个Symfony应用程序： 

# Linux, Mac OS X
$ symfony new my_project_name

# Windows
c:\> cd projects/
c:\projects\> php symfony.phar new my_project_name

该命令创建一个名为my_project_name的新目录，其中包含基于最新稳定的Symfony版本的全新项目。 另外，安装程序会检查您的系统是否符合执行Symfony应用程序的技术要求。 如果没有，您会看到满足这些要求所需的更改列表。

小技巧

出于安全原因，所有Symfony版本在分发之前都进行了数字签名。 如果您想验证任何Symfony版本的完整性，请按照本文中介绍的步骤操作。

基于某个Symfony版本来创建应用¶

如果您的项目需要基于特定的Symfony版本，请将版本号作为新命令的第二个参数传递： 

# Linux, Mac OS X
$ symfony new my_project_name 2.3.23

# Windows
c:\projects\> php symfony.phar new my_project_name 2.3.23

如果您希望您的项目基于最新的Symfony LTS版本，请将lts作为新命令的第二个参数： 

# Linux, Mac OS X
$ symfony new my_project_name lts

# Windows
c:\projects\> php symfony.phar new my_project_name lts

阅读 Symfony发布流程以更好地理解为什么有几个Symfony版本以及哪个版本用于您的项目。

不使用安装工具创建应用¶

如果您仍然使用PHP 5.3，或者因为任何原因无法执行安装程序，则可以使用基于Composer的替代安装方法创建Symfony应用程序。

Composer是现代PHP应用程序使用的依赖管理器，它也可以用来创建基于Symfony框架的新应用程序。 如果您没有在全局安装它，请先阅读下一节。

全局安装Composer¶

从全局安装Composer开始。

通过Composer安装Symfony应用¶

将Composer安装到您的计算机上后，执行create-project命令，基于其最新的稳定版本创建新的Symfony应用程序： 

$ composer create-project symfony/framework-standard-edition my_project_name

如果您需要将应用程序基于特定的Symfony版本，请将该版本作为create-project命令的第二个参数提供： 

$ composer create-project symfony/framework-standard-edition my_project_name "2.3.*"

小技巧

如果您的互联网连接速度很慢，您可能会认为Composer没有做任何事情。 如果这是您的情况，请将-vvv标志添加到上一个命令以显示Composer正在执行的所有操作的详细输出。

运行Symfony应用¶

Symfony利用PHP提供的内部Web服务器在开发应用程序时运行它们。 因此，运行Symfony应用程序需要浏览项目目录并执行以下命令： 

$ cd my_project_name/
$ php app/console server:run

然后，打开浏览器并访问http://localhost:8000URL以查看Symfony的欢迎页面：

您可能会看到空白页面或错误页面，而不是欢迎页面。 这是由目录权限配置错误引起的。 根据您的操作系统，有几种可能的解决方案。 所有这些都在设置权限部分进行了解释。

注解

PHP的内部Web服务器可用于PHP 5.4或更高版本。 如果您仍然使用旧版PHP 5.3版本，则必须在Web服务器中配置虚拟主机。

server:run命令仅适用于开发应用程序。 为了在生产服务器上运行Symfony应用程序，您必须按照Web服务器的配置中的说明配置Apache或Nginx Web服务器。

当您完成Symfony应用程序的工作时，可以使用server:stop命令停止服务器： 

$ php app/console server:stop

检查Symfony应用的配置和运行环境¶

Symfony应用程序随附可视化服务器配置测试程序，以显示您的环境是否可以使用Symfony。 访问以下URL以检查您的配置： 

http://localhost:8000/config.php

如果有任何问题，请在继续之前对其进行更正。

Setting up Permissions

安装Symfony时的一个常见问题是app/cache和 app/logs目录必须由Web服务器和命令行用户写入。 在UNIX系统上，如果您的Web服务器用户与您的命令行用户不同，您可以尝试以下解决方案之一。

1. 使用CLI和Web服务器的相同用户

在开发环境中，为CLI和Web服务器使用相同的UNIX用户是一种常见做法，因为它在设置新项目时避免了任何这些权限问题。 这可以通过编辑Web服务器配置来完成（例如，通常为Apache的httpd.conf或apache2.conf）并将其用户设置为与CLI用户相同（例如，对于Apache，更新用户和用户组）。

2. 使用ACL的系统支持chmod +a命令

许多系统允许你使用chmod +a命令。 首先尝试一下，如果出现错误 - 请尝试下一个方法。 这使用命令来尝试确定您的Web服务器用户并将其设置为HTTPDUSER： 

$ rm -rf app/cache/*
$ rm -rf app/logs/*

$ HTTPDUSER=`ps aux | grep -E '[a]pache|[h]ttpd|[_]www|[w]ww-data|[n]ginx' | grep -v root | head -1 | cut -d\  -f1`
$ sudo chmod +a "$HTTPDUSER allow delete,write,append,file_inherit,directory_inherit" app/cache app/logs
$ sudo chmod +a "`whoami` allow delete,write,append,file_inherit,directory_inherit" app/cache app/logs

3. 使用ACL的系统不支持chmod +a命令

有些系统不支持chmod +a，但支持另一个名为setfacl的实用程序。 您可能需要在分区上启用enable ACL 支持，并在使用之前安装setfacl（与Ubuntu相同）。 这使用命令来尝试确定您的Web服务器用户并将其设置为 HTTPDUSER： 

$ HTTPDUSER=`ps aux | grep -E '[a]pache|[h]ttpd|[_]www|[w]ww-data|[n]ginx' | grep -v root | head -1 | cut -d\  -f1`
$ sudo setfacl -R -m u:"$HTTPDUSER":rwX -m u:`whoami`:rwX app/cache app/logs
$ sudo setfacl -dR -m u:"$HTTPDUSER":rwX -m u:`whoami`:rwX app/cache app/logs

如果这不起作用，请尝试添加-n选项。

4. 不使用ACL

如果前面的方法都不适用于您，请更改umask，以便缓存和日志目录可以分组写入或全局可写（取决于Web服务器用户和命令行用户是否在同一组中）。 为了达到这个目的，在app/console，web/app.php和web/app_dev.php文件的开始放置以下行： 

umask(0002); // This will let the permissions be 0775

// or

umask(0000); // This will let the permissions be 0777

请注意，当您在服务器上访问ACL时，建议使用ACL，因为更改umask不是线程安全的。

Symfony应用升级¶

此时，您已经创建了一个功能齐全的Symfony应用程序，您将在其中开始开发自己的项目。 Symfony应用程序依赖于许多外部库。 这些被下载到vendor/目录中，并由Composer专门管理。

频繁更新这些第三方库是防止错误和安全漏洞的良好做法。 执行更新Composer命令以一次全部更新它们： 

$ cd my_project_name/
$ composer update

根据项目的复杂程度，此更新过程可能需要几分钟才能完成。

小技巧

Symfony提供了一个命令来检查项目的依赖关系是否包含任何已知的安全漏洞：

$ php app/console security:check

一个好的安全措施是定期执行这个命令，以便能够尽快更新或替换受损的依赖关系。

安装Symfony发行版¶

Symfony项目包“发行版”，这些发行版是全功能的应用程序，其中包括Symfony核心库，一系列有用的软件包，合理的目录结构和一些默认配置。 实际上，当您在前面的章节中创建Symfony应用程序时，实际上是下载了由Symfony提供的默认分配，该分配称为Symfony标准版。

Symfony标准版是迄今为止最受欢迎的发行版，它也是开始使用Symfony的开发者的最佳选择。 但是，Symfony社区已经发布了其他可以在您的应用程序中使用的流行发行版：

  • Symfony CMF标准版是开始使用Symfony CMF项目的最佳发行版，该项目使开发人员可以更轻松地将CMS功能添加到使用Symfony框架构建的应用程序中。
  • Symfony REST Edition 展示了如何构建一个应用程序，该应用程序使用FOSRestBundle和其他几个相关软件包来提供RESTful API。

使用代码管理工具¶

如果您使用Git等版本控制系统，则可以安全地提交所有项目的代码。 原因是Symfony应用程序已经包含专门为Symfony准备的.gitignore文件。

有关如何最好地将项目设置为存储在Git中的具体说明，请参阅如何在Git中创建和存储Symfony项目。

检出Symfony应用¶

使用Composer管理应用程序的依赖关系时，建议在将代码提交到存储库之前忽略整个vendor/目录。 这意味着，当从Git存储库中检出Symfony应用程序时，将不会有vendor/目录，并且该应用程序不能立即使用。

为了使其工作，请检查Symfony应用程序，然后执行install Composer命令以下载并安装应用程序所需的所有依赖项： 

$ cd my_project_name/
$ composer install

Composer如何知道要安装哪些特定的依赖关系？ 因为当Symfony应用程序被提交到存储库时，composer.json和composer.lock文件也被提交。 这些文件告诉Composer为应用程序安装哪些依赖项（以及哪些特定版本）。

开始开发工作¶

现在您已经拥有了功能完整的Symfony应用程序，您可以开始开发！ 您的发行版可能包含一些示例代码 - 查看发行版中包含的README.md文件（将其作为文本文件打开），以了解您的发行版包含哪些示例代码。

如果您是Symfony的新手，请查看“创建页面”，在这里您将学习如何创建页面，更改配置以及在新应用程序中执行其他所有操作。

请务必查看手册，其中包含有关解决Symfony特定问题的各种文章。

注解

如果您想从发行版中删除示例代码，请查看本手册文章：“如何删除AcmeDemoBundle”
