控制器（Controller）¶

控制器是一个可调用的PHP，您可以从HTTP请求获取信息并构造并返回HTTP响应（作为Symfony Response对象）。 响应可能是一个HTML页面，一个XML文档，一个序列化的JSON数组，一个图像，一个重定向，一个404错误或者其他你可以想象的东西。 控制器包含应用程序呈现页面内容所需的任意逻辑。

通过查看Symfony控制器，看看这是多么简单。 这呈现出一个打印着名的Hello world!的页面： 

use Symfony\Component\HttpFoundation\Response;

public function helloAction()
{
    return new Response('Hello world!');
}

控制器的目标总是相同的：创建并返回一个Response对象。 一路上，它可能会从请求中读取信息，加载数据库资源，发送电子邮件或设置用户会话的信息。 但在任何情况下，控制器最终都会返回将传递回客户端的Response对象。

没有魔法，也没有其他需要担心！ 以下是一些常见的例子：

  • 控制器A准备一个表示网站主页内容的Response对象。
  • 控制器B从请求中读取slug参数以从数据库加载博客条目并创建一个显示该博客的Response对象。 如果在数据库中找不到slug，它会创建并返回一个带有404状态码的Response对象。
  • 控制器C处理联系表单的表单提交。 它从请求中读取表单信息，将联系信息保存到数据库并将联系信息通过电子邮件发送给您。 最后，它创建一个Response对象，将客户端的浏览器重定向到联系表单“谢谢”页面。

Requests->Controller->Response的过程¶

Symfony项目处理的每个请求都经历了相同的简单生命周期。 框架负责处理所有重复的事情：您只需在控制器函数中编写自定义代码：

  1. 每个请求由一个引导应用程序的单个前端控制器文件（例如 app.php或app_dev.php）处理;
  2. Router从请求中读取信息（例如URI），找到与该信息匹配的路由，并从路由中读取_controller参数;
  3. 执行匹配路由中的控制器，控制器内部的代码创建并返回一个Response对象;
  4. Response对象的HTTP标头和内容被发送回客户端。

创建页面与创建控制器（＃3）以及制作将URL映射到该控制器（＃2）的路线一样简单。

注解

虽然同样命名，但“前端控制器”与本章中讨论的“控制器”不同。 前端控制器是一个简短的PHP文件，它存在于您的Web目录中，并且所有请求都是通过它来执行的。 典型的应用程序将有一个生产前端控制器（例如app.php）和一个开发前端控制器（例如app_dev.php）。 您可能永远不需要编辑，查看或担心应用程序中的前端控制器。

简单示例¶

虽然控制器可以是任何PHP可调用的（函数，对象的方法或Closure），但控制器通常是控制器类中的一种方法。 控制器也被称为actions。 

// src/AppBundle/Controller/HelloController.php
namespace AppBundle\Controller;

use Symfony\Component\HttpFoundation\Response;

class HelloController
{
    public function indexAction($name)
    {
        return new Response('<html><body>Hello '.$name.'!</body></html>');
    }
}

小技巧

请注意，控制器是位于控制器类（HelloController）内的indexAction方法。 不要被命名混淆：控制器类只是将多个控制器/动作组合在一起的简便方法。 通常，控制器类将容纳若干控制器/动作（例如，updateAction, deleteAction等）。

这个控制器非常简单：

  • 第4行：Symfony利用PHP的命名空间功能来命名整个控制器类。 use关键字导入控制器必须返回的Response类。
  • 第6行：类名称是控制器类（即Hello）的名称和单词Controller的串联。 这是一个为控制器提供一致性的约定，并允许它们仅由路由配置中名称的第一部分（即Hello）引用。
  • 第8行：控制器类中的每个操作都以Action作为后缀，并在路由配置中通过操作的名称（index）引用。 在下一节中，您将创建一个将URI映射到此操作的路由。 您将了解路由的占位符（{name}）如何成为操作方法（$name）的参数。
  • 第10行：控制器创建并返回一个Response对象。

创建URL->Controller的映射关系¶

新的控制器返回一个简单的HTML页面。 要在您的浏览器中实际查看此页面，您需要创建一个路由，它将特定的URL路径映射到控制器：

  • 注释 
    // src/AppBundle/Controller/HelloController.php
namespace AppBundle\Controller;

use Symfony\Component\HttpFoundation\Response;
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

class HelloController
{
    /**
     * @Route("/hello/{name}", name="hello")
     */
    public function indexAction($name)
    {
        return new Response('<html><body>Hello '.$name.'!</body></html>');
    }
}
  • YAML 
    # app/config/routing.yml
hello:
    path:      /hello/{name}
    # uses a special syntax to point to the controller - see note below
    defaults:  { _controller: AppBundle:Hello:index }
  • XML 
    <!-- app/config/routing.xml -->
<?xml version="1.0" encoding="UTF-8" ?>
<routes xmlns="http://symfony.com/schema/routing"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://symfony.com/schema/routing
        http://symfony.com/schema/routing/routing-1.0.xsd">

    <route id="hello" path="/hello/{name}">
        <!-- uses a special syntax to point to the controller - see note below -->
        <default key="_controller">AppBundle:Hello:index</default>
    </route>
</routes>
  • PHP 
    // app/config/routing.php
use Symfony\Component\Routing\Route;
use Symfony\Component\Routing\RouteCollection;

$collection = new RouteCollection();
$collection->add('hello', new Route('/hello/{name}', array(
    // uses a special syntax to point to the controller - see note below
    '_controller' => 'AppBundle:Hello:index',
)));

return $collection;

现在，您可以转到/hello/ryan（例如，如果您使用内置 Web服务器，http://localhost:8000/app_dev.php/hello/ryan），Symfony将执行HelloController::indexAction() 控制器并为$name变量传入ryan。 创建一个“页面”意味着简单地创建一个控制器方法和一个关联的路线。

很简单，对吧？

AppBundle：Hello：索引控制器语法

如果您使用YML或XML格式，则可以使用特殊的快捷键语法引用控制器：AppBundle:Hello:index。 有关控制器格式的更多详细信息，请参阅控制器命名规则。

参见

您可以在 路由章节中了解更多有关路由系统的信息。

使用Route参数作为Controller参数¶

您已经知道该路线指向位于AppBundle内部的HelloController::indexAction()方法。 更有趣的是传递给该方法的参数： 

// src/AppBundle/Controller/HelloController.php
// ...
use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

/**
 * @Route("/hello/{name}", name="hello")
 */
public function indexAction($name)
{
    // ...
}

控制器有一个参数$name，它对应于匹配路由中的{name}参数（如果转到/hello/ryan，则返回ryan）。 在执行你的控制器时，Symfony会将每个参数与路由中的一个参数进行匹配。 所以{name}的值被传递给$name。

以下面更有趣的例子：

  • 注释 
    // src/AppBundle/Controller/HelloController.php
// ...

use Sensio\Bundle\FrameworkExtraBundle\Configuration\Route;

class HelloController
{
    /**
     * @Route("/hello/{firstName}/{lastName}", name="hello")
     */
    public function indexAction($firstName, $lastName)
    {
        // ...
    }
}
  • YAML 
    # app/config/routing.yml
hello:
    path:      /hello/{firstName}/{lastName}
    defaults:  { _controller: AppBundle:Hello:index }
  • XML 
    <!-- app/config/routing.xml -->
<?xml version="1.0" encoding="UTF-8" ?>
<routes xmlns="http://symfony.com/schema/routing"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://symfony.com/schema/routing
        http://symfony.com/schema/routing/routing-1.0.xsd">

    <route id="hello" path="/hello/{firstName}/{lastName}">
        <default key="_controller">AppBundle:Hello:index</default>
    </route>
</routes>
  • PHP 
    // app/config/routing.php
use Symfony\Component\Routing\Route;
use Symfony\Component\Routing\RouteCollection;

$collection = new RouteCollection();
$collection->add('hello', new Route('/hello/{firstName}/{lastName}', array(
    '_controller' => 'AppBundle:Hello:index',
)));

return $collection;

现在，控制器可以有两个参数：

public function indexAction($firstName, $lastName)
{
    // ...
}

将路由参数映射到控制器参数非常简单且灵活。 开发时请牢记以下指导原则。

  • 控制器参数的顺序无关紧要

    Symfony将路由中的参数名称与控制器的变量名称进行匹配。 控制器的参数可以完全重新排序并且仍然完美： 

    public function indexAction($lastName, $firstName)
{
    // ...
}

  • 每个必需的控制器参数必须与路由参数匹配

    以下内容会引发RuntimeException，因为路由中没有定义foo参数： 

    public function indexAction($firstName, $lastName, $foo)
{
    // ...
}

    但是，使参数可选是完全正确的。 以下示例不会引发异常：

    public function indexAction($firstName, $lastName, $foo = 'bar')
{
    // ...
}

  • 并非所有的路由参数都需要作为控制器的参数

    例如，如果lastName对您的控制器不重要，则可以完全省略它： 

    public function indexAction($firstName)
{
    // ...
}

小技巧

每条路线还有一个特殊的_route参数，该参数等于匹配的路线的名称（例如hello）。 虽然通常不会有用，但这也可以作为控制器参数。 您还可以将路线中的其他变量传递给控制器参数。 请参阅如何将额外信息从路由传递到控制器。

使用 Request 对象作为Controller参数¶

如果您需要阅读查询参数，获取请求标题或访问上传的文件，该怎么办？ 所有这些信息都存储在Symfony的Request对象中。 要在控制器中获取它，只需将它作为参数添加并 - 键入使用Request类提示它： 

use Symfony\Component\HttpFoundation\Request;

public function indexAction($firstName, $lastName, Request $request)
{
    $page = $request->query->get('page', 1);

    // ...
}

参见

想要了解更多有关从请求中获取信息的信息？ 请参阅 访问请求信息。

Controller基类¶

为了方便起见，Symfony带有一个可选的基本Controller类。 如果扩展它，您将通过容器访问许多辅助方法和所有服务对象（请参阅访问其他Services）。

在Controller类上添加use语句，然后修改HelloController以扩展它： 

// src/AppBundle/Controller/HelloController.php
namespace AppBundle\Controller;

use Symfony\Bundle\FrameworkBundle\Controller\Controller;

class HelloController extends Controller
{
    // ...
}

这实际上并没有改变你的控制器的工作方式：它只是让你访问基本Controller类可用的帮助器方法。 这些只是使用核心Symfony功能的快捷方式，无论是否使用基本Controller类，都可以使用这些功能。 查看核心功能的好方法是查看Controller 类。

参见

如果您对控制器如何工作并没有扩展这个基类感到好奇，请查看控制器作为服务。 这是可选的，但可以让您更好地控制注入控制器的确切对象/依赖关系。

重定向¶

如果您想将用户重定向到另一个页面，请使用redirectToRoute()方法： 

public function indexAction()
{
    return $this->redirectToRoute('homepage');

    // redirectToRoute is equivalent to using redirect() and generateUrl() together:
    // return $this->redirect($this->generateUrl('homepage'), 301);
}

2.6 新版功能: redirectToRoute()方法在Symfony 2.6中添加。 以前（现在仍然是），您可以一起使用redirect()和generateUrl()（请参阅上面的示例）。

或者，如果您想要外部重定向，只需使用redirect()并将URL传递给它： 

public function indexAction()
{
    return $this->redirect('http://symfony.com/doc');
}

默认情况下，redirectToRoute()方法执行302（临时）重定向。 要执行301（永久）重定向，请修改第三个参数： 

public function indexAction()
{
    return $this->redirectToRoute('homepage', array(), 301);
}

小技巧

redirectToRoute()方法只是创建专门用于重定向用户的Response对象的快捷方式。 这相当于： 

use Symfony\Component\HttpFoundation\RedirectResponse;

public function indexAction()
{
    return new RedirectResponse($this->generateUrl('homepage'));
}

模板渲染¶

如果您正在提供HTML，则需要呈现模板。 render()方法呈现模板并将该内容放置到您的Response对象中： 

// renders app/Resources/views/hello/index.html.twig
return $this->render('hello/index.html.twig', array('name' => $name));

您也可以将模板放在更深的子目录中。 尽量避免造成不必要的深层结构：

// renders app/Resources/views/hello/greetings/index.html.twig
return $this->render('hello/greetings/index.html.twig', array('name' => $name));

Symfony模板引擎在模板章节中有详细的介绍。

引用包含在包内的模板

您还可以将模板放入捆绑包的Resources/views目录中，并使用BundleName:DirectoryName:FileName语法引用它们。 例如，AppBundle:Hello:index.html.twig将引用位于src/AppBundle/Resources/views/Hello/index.html.twig中的模板。 请参阅引用Bundle里的模板。

访问其他Services¶

Symfony包含了很多有用的对象，称为服务。 这些用于呈现模板，发送电子邮件，查询数据库以及您可以想到的任何其他“工作”。 当你安装一个新的软件包时，它可能会带来更多的服务。

扩展基础控制器类时，可以通过get()方法访问任何Symfony服务。 以下是您可能需要的几项常用服务： 

$templating = $this->get('templating');

$router = $this->get('router');

$mailer = $this->get('mailer');

还有哪些其他服务？ 您可以列出所有服务，请使用debug:container命令： 

$ php app/console debug:container

2.6 新版功能: 在Symfony 2.6之前，这个命令被称为container:debug。

有关更多信息，请参阅服务容器（Service Container）一章。

生成404或其他错误页面¶

当事情没有找到时，你应该和HTTP协议一起玩，并返回一个404响应。 为此，您将抛出特殊类型的异常。 如果您要扩展基础控制器类，请执行以下操作：

public function indexAction()
{
    // retrieve the object from database
    $product = ...;
    if (!$product) {
        throw $this->createNotFoundException('The product does not exist');
    }

    return $this->render(...);
}

createNotFoundException()方法只是创建一个特殊的NotFoundHttpException对象的快捷方式，它最终触发Symfony中的404 HTTP响应。

当然，你可以自由地在控制器中抛出任何Exception类--Symfony会自动返回500个HTTP响应代码。 

throw new \Exception('Something went wrong!');

在任何情况下，最终用户都会看到一个错误页面，并向开发人员显示完整的调试错误页面（即，当您使用app_dev.php时 - 请参阅运行环境 & Front Controllers。

您需要自定义用户看到的错误页面。 要做到这一点，请参阅“如何自定义错误页面”

Session管理¶

Symfony提供了一个很好的会话对象，您可以使用它来在请求之间存储有关用户的信息（无论是使用浏览器，机器人还是Web服务的真人）。 默认情况下，Symfony通过使用本地PHP会话将属性存储在cookie中。

从任何控制器可轻松实现从会话存储和检索信息：

use Symfony\Component\HttpFoundation\Request;

public function indexAction(Request $request)
{
    $session = $request->getSession();

    // store an attribute for reuse during a later user request
    $session->set('foo', 'bar');

    // get the attribute set by another controller in another request
    $foobar = $session->get('foobar');

    // use a default value if the attribute doesn't exist
    $filters = $session->get('filters', array());
}

这些属性将保留在该用户会话剩余部分的用户身上。

Flash消息¶

您还可以存储将存储在用户会话中的小消息，以准确地追加一个请求。 处理表单时这很有用：您想重定向并在下一页显示特殊消息。 这些类型的消息被称为“闪存”消息。.

例如，假设您正在处理表单提交：

use Symfony\Component\HttpFoundation\Request;

public function updateAction(Request $request)
{
    $form = $this->createForm(...);

    $form->handleRequest($request);

    if ($form->isValid()) {
        // do some sort of processing

        $this->addFlash(
            'notice',
            'Your changes were saved!'
        );

        // $this->addFlash is equivalent to $this->get('session')->getFlashBag()->add

        return $this->redirectToRoute(...);
    }

    return $this->render(...);
}

处理完请求后，控制器会在会话中设置通知 闪烁消息，然后重定向。 名称（通知）并不重要 - 这只是您发明和参考下一个。

在下一个操作的模板中，可以使用以下代码来呈现通知消息：

  • Twig 
    {% for flashMessage in app.session.flashbag.get('notice') %}
    <div class="flash-notice">
        {{ flashMessage }}
    </div>
{% endfor %}
  • PHP 
    <?php foreach ($view['session']->getFlash('notice') as $message): ?>
    <div class="flash-notice">
        <?php echo "<div class='flash-error'>$message</div>" ?>
    </div>
<?php endforeach ?>

在下一个操作的模板中，可以使用以下代码来呈现通知消息：根据设计，Flash消息只能用于一个请求（它们“一闪而过”）。 它们被设计用于跨越重定向，就像你在这个例子中所做的一样。

Response对象¶

控制器的唯一要求是返回一个Response对象。 Response类是HTTP响应的抽象：基于文本的消息充满了发送回客户端的标题和内容： 

use Symfony\Component\HttpFoundation\Response;

// create a simple Response with a 200 status code (the default)
$response = new Response('Hello '.$name, Response::HTTP_OK);

// create a JSON-response with a 200 status code
$response = new Response(json_encode(array('name' => $name)));
$response->headers->set('Content-Type', 'application/json');

headers属性是一个HeaderBag对象，并有一些用于获取和设置标题的好方法。 标题名称已标准化，因此使用Content-Type相当于content-type或甚至content_type。

还有特殊的类可以使某些类型的反应更容易：

  • 对于JSON，有JsonResponse。 请参阅创建JSON响应。
  • 对于文件，有BinaryFileResponse。 请参阅服务文件。
  • 对于流式响应，有 StreamedResponse。 请参阅流式传输响应。

参见

别担心！ 有关组件文档中的Response对象的更多信息。 请参阅Response.。

Request对象¶

除了路由占位符的值之外，控制器还可以访问Request对象。 如果某个变量是用请求进行了类型提示的，那么框架将在控制器中注入Request对象： 

use Symfony\Component\HttpFoundation\Request;

public function indexAction(Request $request)
{
    $request->isXmlHttpRequest(); // is it an Ajax request?

    $request->getPreferredLanguage(array('en', 'fr'));

    $request->query->get('page'); // get a $_GET parameter

    $request->request->get('page'); // get a $_POST parameter
}

与Response对象一样，请求标头存储在HeaderBag对象中，并且很容易访问。

参见

别担心！ 有关组件文档中Request对象的更多信息。 请参阅请求.。

Forward到其他Controller¶

尽管不是很常见，但您也可以使用forward()方法在内部转发到其他控制器。 它不是重定向用户的浏览器，而是创建一个内部的子请求，并调用控制器。forward()方法返回从该控制器返回的Response对象： 

public function indexAction($name)
{
    $response = $this->forward('AppBundle:Something:fancy', array(
        'name'  => $name,
        'color' => 'green',
    ));

    // ... further modify the response or return it directly

    return $response;
}

请注意，forward()方法使用控制器的特殊字符串表示形式（请参阅Controller命名规则）。 在这种情况下，目标控制器函数将是AppBundle内的SomethingController::fancyAction()。 传递给该方法的数组成为生成的控制器上的参数。 将控制器嵌入到模板中时使用了相同的想法（请参见嵌入Controller）。 目标控制器方法看起来像这样： 

public function fancyAction($name, $color)
{
    // ... create and return a Response object
}

就像为路由创建控制器一样，fancyAction的参数顺序并不重要。 Symfony将索引键名称（例如name）与方法参数名称（例如$name）进行匹配。 如果您更改参数的顺序，Symfony仍然会将正确的值传递给每个变量。

总结¶

无论何时创建页面，最终都需要编写一些包含该页面逻辑的代码。 在Symfony中，这称为控制器，它是一个PHP函数，您可以执行任何操作以返回将返回给用户的最终Response对象。

为了使生活更轻松，您可以选择扩展基本控制器，其中包含许多常见控制器任务的快捷方法。 例如，由于您不想在控制器中放置HTML代码，因此可以使用render()方法呈现并返回模板中的内容。

在其他章节中，您将看到如何使用控制器持久存取数据库中的对象，处理表单提交，处理缓存等等。
