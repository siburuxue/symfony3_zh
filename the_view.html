视图¶

在阅读本教程第一部分之后，你已经决定继续读下去。 在第二部分你将会学习Twig：快速、灵活并且安全的用于PHP应用程序的模板引擎。 Twig是您的模板更具有可读性和简洁性，同时也使得他们更适合Web应用程序。

熟悉Twig¶

Twig官方文档是了解有关此模板引擎的最佳资源。 这部分只是介绍其主要概念。

Twig是一个文本文件可以生成任何类型的内容(HTML, CSS, JavaScript, XML, CSV, LaTeX, 等)。使用下面的任何一个分隔符都可以将Twig元素和其他的模板内容分割开：

{{ ... }}
显示变量的内容或者表达式的结果。
{% ... %}
用来控制模板的逻辑例如for循环和if条件判断。
{# ... #}
允许模板中包含注释。与html注释相反，他们不会包含在渲染模板中。

下面是一个最简单的模板，延时了一些基本的用法，使用两个变量page_title和navigation，他们将被传递到模板中： 

<!DOCTYPE html>
<html>
    <head>
        <title>{{ page_title }}</title>
    </head>
    <body>
        <h1>{{ page_title }}</h1>

        <ul id="navigation">
            {% for item in navigation %}
                <li><a href="{{ item.url }}">{{ item.label }}</a></li>
            {% endfor %}
        </ul>
    </body>
</html>

要在Symfony中渲染模板，使用控制器内的render方法。如果模板需要变量生成内容，第二个可选参数传入数组： 

$this->render('default/index.html.twig', array(
    'variable_name' => 'variable_value',
));

传递给模板的变量可以是字符串，数组，甚至是对象。Twig区分他们之间的差异，可以让你使用点符号（.）来访问一个变量的属性。 下面的代码清单展示了如何根据控制器传递的变量类型来显示变量的内容： 

{# 1. Simple variables #}
{# $this->render('template.html.twig', array('name' => 'Fabien') ) #}
{{ name }}

{# 2. Arrays #}
{# $this->render('template.html.twig', array('user' => array('name' => 'Fabien')) ) #}
{{ user.name }}

{# alternative syntax for arrays #}
{{ user['name'] }}

{# 3. Objects #}
{# $this->render('template.html.twig', array('user' => new User('Fabien')) ) #}
{{ user.name }}
{{ user.getName }}

{# alternative syntax for objects #}
{{ user.name() }}
{{ user.getName() }}

装饰模板¶

在通常情况下，模板会共享公共元素，比如总所周知的页眉和页脚。Twig用一个叫模板继承的概念优雅的解决这个问题。该功能允许您创建一个基本模板，其中包含你网站的所有常用的元素，并定义子模板可以覆盖的内容“block”。

index.html.twig模板使用extends标签来指示他从base.html.twig模板继承: 

{# app/Resources/views/default/index.html.twig #}
{% extends 'base.html.twig' %}

{% block body %}
    <h1>Welcome to Symfony!</h1>
{% endblock %}

打开base.html.twig模板对应的app/Resources/views/base.html.twig文件，你将找到一下Twig代码： 

{# app/Resources/views/base.html.twig #}
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <title>{% block title %}Welcome!{% endblock %}</title>
        {% block stylesheets %}{% endblock %}
        <link rel="icon" type="image/x-icon" href="{{ asset('favicon.ico') }}" />
    </head>
    <body>
        {% block body %}{% endblock %}
        {% block javascripts %}{% endblock %}
    </body>
</html>

{% block %}标签告诉模板引擎，子模板可能会覆盖模板的哪些部分。 在这个例子中，index.html.twig模板覆盖body block，而不是标题block，他讲显示在base.html.twig模板中定义的默认内容。

使用标签，过滤器和方法¶

Twig的最佳特征之一是使用标签，过滤器和方法的可扩展性。看看下面这个示例模板，广泛使用过滤器来修改信息然后在显示给用户： 

<h1>{{ article.title|capitalize }}</h1>

<p>{{ article.content|striptags|slice(0, 255) }} ...</p>

<p>Tags: {{ article.tags|sort|join(", ") }}</p>

<p>Activate your account before {{ 'next Monday'|date('M j, Y') }}</p>

不要忘了查看官方的Twig文档来了解有关过滤器，函数和标签的所有信息。

包含其他模板¶

在多个模板之间共享代码片段的最佳方式是创建一个新的模板片段，然后可以从其他模板中包含这个模板片段。

想象一下，我们希望在我们的应用程序的某些页面上展示广告。第一步，创建一个banner.html.twig模板： 

{# app/Resources/views/ads/banner.html.twig #}
<div id="ad-banner">
    ...
</div>

要在任何一个页面上显示此广告，请使用include()方法包含banner.html.twig模板： 

{# app/Resources/views/default/index.html.twig #}
{% extends 'base.html.twig' %}

{% block body %}
    <h1>Welcome to Symfony!</h1>

    {{ include('ads/banner.html.twig') }}
{% endblock %}

嵌入其他控制器¶

如果要将另一个控制器的结果嵌入到模板中该怎么办？这在使用ajax或者当嵌入模板需要一些在主模板中不可见的变量时非常有用。

假设你已经创建了一个topArticlesAction控制器方法来显示你网站上最受欢迎的文章。 如果要在index模板中呈现该方法的结果（通常是一些html内容），请使用render()函数： 

{# app/Resources/views/index.html.twig #}
{{ render(controller('AppBundle:Default:topArticles')) }}

在这里，render() 和controller()方法 使用特殊的AppBundle:Default:topArticles 语法来引用Default 控制器中的topArticlesAction action （AppBundle部分将在稍后解释）： 

// src/AppBundle/Controller/DefaultController.php

class DefaultController extends Controller
{
    public function topArticlesAction()
    {
        // look for the most popular articles in the database
        $articles = ...;

        return $this->render('default/top_articles.html.twig', array(
            'articles' => $articles,
        ));
    }

    // ...
}

创建页面间的链接¶

创建页面间的链接是web应用程序的必备条件。 path函数不是在模板中硬编码生成URL，而是知道如何根据路由配置生成URl。这样只要更新配置即可更新所有网址： 

<a href="{{ path('homepage') }}">Return to homepage</a>

path函数以路由名作为第一个参数，您可以选择传入一个路由数组作为第二个参数

小技巧

url函数和path函数是非常相似的，生成绝对路径，尤其在渲染邮件和RSS文件时时非常方便的： <a href="{{ url('homepage') }}">Visit our website</a>.

包含资源文件: Images, JavaScripts 和 Stylesheets¶

如果互联网没有images,javascripts和stylesheets会是什么样子？ Symfony提供了asset方法，可以很容易的处理他们： 

<link href="{{ asset('css/blog.css') }}" rel="stylesheet" type="text/css" />

<img src="{{ asset('images/logo.png') }}" />

asset()方法在web/目录中查找资源文件。 如果您将他们存储在另一个目录，阅读这篇文章以了解如何管理web资源文件。

使用asset方法，您的应用程序变得更便捷。 原因是您可以将应用程序的根目录移到您的web根目录下的任何位置，而无需更改模板代码中的任何内容。

最后的想法¶

Twig简单而强大。感谢它包含了layouts, blocks, templates and action，th非常容易的以扩及和可扩展的方式组织模板。

你只用了Symfony20分钟，但你已经可以用它做非常了不起的事情了。这就是Symfony的强大。学习基本知识是很容易的，你很快就会发现这种简单性隐藏在一个非常灵活的体系结构下。

但我正在超越自己。首选，您需要了解更多关于控制器的信息，这正是下一个部分的主题。准备好继续深入学习了么？
